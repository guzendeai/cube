# CLAUDE.md — Claude Code 作業ルール

このファイルは、Claude Code がこのリポジトリで作業するときのルールと文脈を記録したものです。

## 基本ルール

- **いきなり大規模変更しない。** 作業前に現在の構成を確認し、変更するファイルと理由を説明してから実装する。
- **既存コードを確認してから変更する。** ファイルを上書きする前に必ず Read ツールで内容を読む。
- **不要な削除をしない。** コード・データ・画像・リンク・設定は、用途が確認できるまで削除しない。
- **段階的に変更する。** 一度に全体を完成させようとせず、フェーズごとに進める。

## 技術ルール

- TypeScript エラーを残さない。`any` の多用を避ける。
- スマホ表示（375px〜）を必ず考慮する。`100dvh` を使う。
- `clamp()` で文字サイズや余白をレスポンシブにする。
- 重すぎる 3D 表現・ライブラリを導入しない（canvas + CSS で十分）。

## デザインルール

- このサイトのデザイン方針は `DESIGN.md` を参照。
- このサイトのコンセプトは `CONCEPT.md` を参照。
- デザインの意図（静けさ・余白・暗さ）を壊さない。
- 派手な UI ライブラリや企業サイト的なコンポーネントを使わない。

## ファイル構成（概要）

```
src/
  app/
    page.tsx           ← トップページ（点が浮かぶ空間）
    contents/          ← コンテンツ一覧（既存・変更不要）
    words/             ← 言葉・本（既存・変更不要）
    stage/             ← ステージ演出（既存・変更不要）
  components/
    CategoryPanel.tsx  ← 6カテゴリの詳細パネル
    Cube3D.tsx         ← 旧キューブ UI（保持・非使用中）
    ContentModal.tsx   ← 旧モーダル（保持・非使用中）
  data/
    siteData.ts        ← 6カテゴリのメタ＋仮データ
    contents.ts        ← 旧コンテンツデータ（保持）
    words.ts           ← 言葉データ（保持）
  types/
    index.ts           ← 6カテゴリの TypeScript 型
```
